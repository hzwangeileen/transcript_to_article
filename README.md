# 转录稿编译为文章 Skill

将飞书妙记或其他工具导出的语音转文字稿，编译为结构完整的中文文章，输出为 Word 文档（.docx）。

## 适用场景

- 活动讨论、圆桌会议的记录整理
- 一对一或多人访谈的内容编译
- 演讲或分享会的文字整理

转录稿支持中文或英文，说话人可标注为"说话人1""说话人2"或直接标注人名。

## 工作流程

### 第一步：结构预览（等待确认）

上传转录稿后，Skill 会输出以下三项内容，然后暂停等待用户确认：

1. **转录错误修正清单**：以表格形式列出所有识别到的转录错误（人名、公司名、产品名、专业术语等），格式为"原文 → 修正后"。
2. **章节结构**：围绕核心观点重新组织，归纳为 5 到 8 章，每章附一句话说明。章节按逻辑顺序排列，不按对话顺序划分。
3. **文风示范**：直接用目标文风写一段约 100 到 150 字的正文示例，并询问是否需要加入作者判断视角。

### 第二步：生成完整 Word 文档

用户确认结构和文风后，直接生成完整 .docx 文件，不再逐章停顿。

文档结构依次为：
- 标题备选（3 到 5 个，风格各异）
- 摘要金句（2 到 3 句，适合放在标题下方）
- 引言（200 到 300 字，带紫色边框，末尾附核心观点列表）
- 正文（按章节编号展开）

## 输出格式

- **章节标题**：编号 `01`、`02`……，紫色编号 + 黑色标题，直接进入正文，无副标题行
- **子标题**：紫色（`#5B4FCF`），论断型或疑问型，不编号
- **正文**：纯文章体，每段 3 到 5 句，金句加粗，无对话痕迹
- **备注框**：浅紫色边框，用于专有名词解释、产品背景、最新动态

## 编译原则

- **信息零损失**：所有内容完整保留，仅删除口语语气词（"嗯""啊""like"等）
- **篇幅对等**：输出字数与原文字数大体相当，不大幅压缩
- **观点归属**：说话人的独家数据、个人判断保留出处标注；公开数据可直接陈述
- **错误修正**：飞书转录错误根据语境和公开知识主动修正，成文中直接使用修正后版本

## 文风说明

默认为研究报告体：结论前置、判断明确、数据与结论紧密绑定，保留说话人归属（如"Cameron 指出"），不加入作者立场语句。

如需对齐特定文风，上传参考文章 PDF 即可，Skill 会自动提取其风格特征并覆盖默认风格。

## 技术说明

使用 `docx` npm 包（JavaScript）生成文件，输出至 `/mnt/user-data/outputs/`。

颜色参数：
| 用途 | 颜色值 |
|------|--------|
| 章节编号 / 子标题 | `#5B4FCF` |
| 备注框背景 | `#F5F3FF` |
| 备注框左边框 | `#9B8FE0` |
| 引言框背景 | `#EDE9FF` |
| 引言框全边框 | `#5B4FCF` |
